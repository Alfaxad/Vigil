Quickstart

Get Vigil auditing your agent's transactions in under 2 minutes.

1. Register your agent

curl -X POST https://vigil.nadhari.ai/api/register \ -H "Content-Type: application/json" \ -d '{"name": "my-trading-bot", "wallet_address": "0xYourAgentWallet"}'

Save your API key immediately. If you lose it, you can regenerate a new one using your operator ID.

2. Audit a transaction

curl -X POST https://vigil.nadhari.ai/api/audit \ -H "Authorization: Bearer vgl_your_api_key" \ -H "Content-Type: application/json" \ -d '{ "transaction": { "from": "0xYourAgent", "to": "0xCounterparty", "amount": 150.00, "memo": "Swap USDC for ETH" }, "agent_context": { "name": "TradingBot", "task": "Execute approved DeFi trades under $500" } }'

3. Read the verdict

Every audit returns a structured response with a verdict, risk score, and reasoning:

verdict:APPROVE
risk_score:0.12
reasoning:"Standard swap to verified Uniswap Router. Counterparty seen 347 times."
attestation_tx:0xabc1...d82f
domain:defi-swap

Authentication

All API requests require a Bearer token in the Authorization header. Your API key is generated when you register and starts with vgl_.

Authorization: Bearer vgl_your_api_key
Lost your key? Use POST /api/regenerate-key with your operator_id to get a new one without re-registering.

Audit Pipeline

Every transaction flows through four layers before a verdict is issued:

Layer 1: Perception

Statistical anomaly detection. Vigil maintains behavioral baselines for each agent and computes z-scores against historical patterns. Deviations in amount, frequency, counterparty, and timing are flagged.

Layer 2: Policy

Natural language rules defined by the guardian. Policies express hard boundaries ("never transfer more than $1000 to unknown addresses") and soft preferences ("prefer verified DEXes for swaps").

Layer 3: Reasoning

Agentic reasoning powered by Claude. Analyzes the transaction in context: does this action make sense given the agent's stated task? Is the counterparty trustworthy? Is the amount proportional to the goal?

Layer 4: Signal

Output and attestation. The verdict, risk score, and reasoning are packaged into an ERC-8004 attestation and written to Base. The counterparty reputation graph is updated.

Policy Engine

Policies are natural language rules that constrain agent behavior. They are evaluated in Layer 2 of the pipeline.

// Example policies "Block any transfer exceeding $5,000 to an unknown address" "Flag NFT purchases above $500 for manual review" "Only allow DeFi swaps on Uniswap, Aerodrome, and Aave" "Require Layer 2 review for first-time counterparties"

Policies can be configured per-guardian through the dashboard's Policy Editor.

ERC-8004 Attestations

Every audit is recorded as a verifiable onchain attestation on Base using the ERC-8004 Skill Protocol. This creates an immutable receipt that any agent in the ecosystem can verify.

Attestation schema: vigil-audit-v1

{ "schema": "vigil-audit-v1", "attester": "vigil.base", "subject": "0xAgentAddress", "verdict": "APPROVE", "risk_score": 0.12, "domain": "defi-swap", "reasoning_hash": "0x...", "timestamp": 1711234567 }

Agent SDK

Integrate Vigil directly into your agent's decision loop.

Python

import requests def audit_before_execute(tx, agent_context): response = requests.post( "https://vigil.nadhari.ai/api/audit", headers={"Authorization": "Bearer vgl_your_key"}, json={ "transaction": tx, "agent_context": agent_context } ) result = response.json() if result["verdict"] == "BLOCK": raise Exception(f"Vigil blocked: {result['reasoning']}") if result["verdict"] == "FLAG": notify_guardian(result) return result

JavaScript / TypeScript

async function auditTransaction(tx, agentContext) { const res = await fetch('https://vigil.nadhari.ai/api/audit', { method: 'POST', headers: { 'Authorization': 'Bearer vgl_your_key', 'Content-Type': 'application/json' }, body: JSON.stringify({ transaction: tx, agent_context: agentContext }) }); const result = await res.json(); if (result.verdict === 'BLOCK') { throw new Error(`Vigil blocked: ${result.reasoning}`); } return result; }

CLI Reference

Vigil ships with a full-featured terminal interface. Every web feature is available from the command line — ideal for agents that operate without a browser.

Install

git clone https://github.com/Alfaxad/Vigil.git cd Vigil pip install -e . # Installs the `vigil` command globally

Authentication

# Register and get an API key (saved to ~/.vigil/config.json) vigil register --name "my-agent" # Point CLI to your remote server vigil config --server https://vigil.nadhari.ai # Regenerate a lost key vigil regenerate-key # Show saved config vigil config --show

Audit

# Standard audit ($0.01) vigil audit \ --from 0xAgentWallet \ --to 0xRecipient \ --amount 150 \ --memo "NFT purchase" \ --agent-name "TradingBot" \ --agent-task "Execute approved purchases under $200" # Deep audit with counterparty profile ($0.05) vigil audit --from 0x... --to 0x... --amount 5000 --deep # JSON output for piping to other tools vigil audit --from 0x... --to 0x... --amount 100 --json-output

Reputation and Monitoring

# Lookup trust score for any address vigil reputation 0x2626664c2603336e57b271c5c0b26f421741e481 # View the full reputation graph vigil graph # Register a wallet for continuous monitoring vigil monitor --wallet 0xAgent --name "SwapBot" --task "DeFi swaps" # List monitored agents vigil agents

Policies

# Add a natural language policy vigil policies add --rule "Block transfers over $5000 to unknown addresses" # List all active policies vigil policies list

Live Monitoring and History

# Live-tail audit results (Ctrl+C to stop) vigil watch # Recent audit history vigil audits --limit 50 # Onchain ERC-8004 attestations vigil attestations # x402 pricing discovery vigil pricing # Server status and wallet balance vigil status

Webhooks

Configure webhooks to receive real-time notifications when audits complete.

curl -X POST https://vigil.nadhari.ai/api/webhooks \ -H "Authorization: Bearer vgl_your_key" \ -d '{ "url": "https://your-app.com/vigil-webhook", "events": ["audit.flag", "audit.block"] }'

Available events

EventFires when
audit.approveTransaction approved
audit.flagTransaction flagged for review
audit.blockTransaction blocked
reputation.changeA counterparty's trust score changes significantly
